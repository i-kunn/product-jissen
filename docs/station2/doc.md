# 要件定義

ここにマークダウン形式のテキストやリンクを記載する。
※リンクの場合は、リンクを知っている全員が閲覧できるように権限設定してください。
① 身の回りの課題
•	アプリを使っていても情報共が分散：休暇明けは各所に分散した申し送りを遡って探す必要があり、読み漏れが発生。
•	紙で運用している：保管・印刷コストが高い。
    必要な情報を探すのに時間がかかる。
　　外出予定・受診予定などの行事・予定情報を忘れてしまうリスクがある
•	最悪の場合、重要情報の見落としが利用者の安全リスクにつながる。

② ターゲット
•	対象施設：特養・老健（60–120床、職員40–100名、3交代制）
•	現状：紙運用 or アプリ併用で申し送りが点在
•	主なユーザー：介護職／看護職／相談員／機能訓練指導
③ 解決策と提供価値
解決策
•	利用者ごとの「申し送りノートの部屋」を作成し、投稿を一元化
•	既読/未読管理。出勤時に前回退勤以降の未読を自動抽出して通知
•	重要度タグ（例：服薬/転倒/発熱）と要約表示で優先度付け
•	タスク（利用者名＋実施時間）を登録し、当日の午前０時にアプリでリマインド
外出予定、受診などの予定を忘れずに確実に行える。
提供できる価値（効果目標）
•	休暇明けの遡り時間：1０分→５分
•	紙関連コスト：年3千600円→0円
•	重要ノート未読率：0%運用
