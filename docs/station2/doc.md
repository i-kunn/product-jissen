介護施設向け申し送りノートアプリ


① 身の回りの課題
•	アプリを使っていても情報共が分散：休暇明けは各所に分散した申し送りを遡って探す必要がありまた申し送りを遡る量が多いため、時間がかかり、読み漏れが発生。
•	紙で運用している：保管・印刷コストが高く、紛失などで情報漏洩の危険がある。

•シフトによる情報断絶
　毎日勤務する職員が異なるため、前日に発生した出来事が翌日の担当に確実に伝わらないケースがある。
　申し送りの抜け漏れにより、利用者対応の質にムラが発生している

•	最悪の場合、重要情報の見落としが利用者の安全。リスクにつながる。


② ターゲット
•	対象施設：特養・老健（60–120床、職員40–100名、3交代制）
•	現状：紙運用 or アプリ併用で申し送りが点在
•	主なユーザー：介護職／看護職／相談員／機能訓練指導

③ 解決策と提供価値

解決策
機能要件
1. 認証機能（ログイン／ログアウト）

社員は 名前（ユーザー名）とパスワード を入力してログインする。

ログインに必要なアカウント情報（名前・パスワード）は 管理者が事前に作成・管理 する。

社員は付与されたアカウントを用いてのみアプリへアクセス可能。

ログイン後は、アプリ内のログアウトボタンから いつでもログアウトできる。

A要求
a-1登録したタスクを忘れないようにしたい

機能要件
2. タスク機能（予定のリマインド）
2-1. タスク登録

「利用者名」と「実施時間」をセットにしてタスクとして登録できる。

ホーム画面では「当日のタスク一覧」を表示する。

ホーム画面からタスクの編集・削除が可能である。編集は担当者が行う。

2-2. 期限判定と表示

タスクの実施時間を過ぎた場合、または日付が翌日の午前0時を過ぎた場合、タスクを「期限切れ」として表示する。

「期限切れ」となったタスクであっても、自動的に消去はされず、翌日になってもホーム画面上に残るものとする

A要求
a-2職員間の情報共有の漏れをなくしたい

機能要件
3. 職員間情報共有内容
3-1. 委員会・行事情報、その他情報の共有

各委員会が担当する業務内容、および施設内の行事予定を職員全員が確認できるようにする。

情報は最新の内容が反映され、更新があった場合は職員が常に把握できる状態とする。

3-2. 職員間部屋情報の作成

現場職員が、新しく必要になった「部屋」をアプリ上で作成できるようにする。

作成した部屋は、全職員が利用できる共有情報として扱われる。

A要求
a-3. 新規入所者情報を安全に管理し、必要な時に確認できるようにしたい

機能要件
4. 新規入所者情報共有機能
4-1. 新規入所者情報の確認

新規入所者の基本情報および必要な共有事項を、職員全員が確認できるようにする。

情報は最新状態が反映され、入所直後の対応に活用できる。

4-2. 利用者部屋作成権限

新規入所者の部屋を作成できるのは「管理者のみ」とする。

一般職員は閲覧のみ可能とし、編集権限は持たない。

4-3. 入所者情報の追加期間

入所者の初期情報の追加・追記が可能なのは「入所から 1 週間以内」に限定する。

1週間経過後は閲覧のみとなり、追加編集は管理者のみが行えるものとする。

A要求
a-4.未読の情報を確実に把握できるようにしたい

機能要件
5. 利用者ごとの「申し送りノートの部屋」

各利用者に 専用の申し送りノート部屋 を作成し、情報を投稿、写真を添付できるようにする。

部屋の作成と名前は管理者のみ作成することができるようにする

検索機能で以下の条件から利用者を検索可能：部屋番号　名前（名）　苗字（姓）　フルネーム　投稿に含まれるコメント（キーワード検索）

6. 未読管理と通知
6-1. 未読の自動抽出

職員が 出勤したタイミングで、前回退勤以降の未読投稿を自動抽出。

抽出した未読情報を一覧、アラート表示で通知し、読み漏れを防止する。

6-2. 未読カレンダー表示

各利用者に 専用カレンダー を表示し、投稿を日付ごとに整理する。

未読がある日は赤色表示 し、読み漏れを防止する。

情報を「日付」単位で集約することで、記録者ごと・投稿者ごとに情報が散らばらず、
必要な申し送りを1つの画面からすぐに確認できる UI を実現する。


非機能要件
1. 認証機能（ログイン／ログアウト）
ログアウト後はログイン画面へ戻り、再度認証情報の入力が必要となる。

ログアウト時には セッション情報を完全に破棄し、安全性を確保する。

パスワードは一定の 最低文字数 を設定し、安全性を担保。


提供できる価値（効果目標）
•	休暇明けの遡り時間：1０分→５分
•	紙関連コスト：年3千600円→0円
•	0％を目指す運用を実現（未読の自動抽出・アラート表示により防止）
